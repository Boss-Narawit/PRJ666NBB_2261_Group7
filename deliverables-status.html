The only deliverable that is genuinely missing, and the only place the app shows fabricated data (a hardcoded 75%). The professor approved a pre-built AI — fashion-clip is exactly that. Everything around it already exists (schema, unique-pair index, env var, thresholds in constants.js):

  • ai-service: minimal FastAPI app on port 8000 with one endpoint — POST /embed (image URL → 512-dim fashion-clip vector). Verify with curl.
  • Backend: similarity.service.js — embed the candidate photo, embed-or-read-cache Clothing.aiEmbedding, cosine-compare against status: 'Available' items (BR19), persist SimilarityCheck docs (BR18), reject when no imageUrl (BR6), emit similarity_alert at ≥ 0.70 (BR16). Replace the stub similarityRoutes.js with a real auth-gated route. Test with the AI service mocked at the HTTP boundary (nock).
  • Frontend: replace the hardcoded 75 in ThoughtfulPurchasingScreen.tsx:41 with the real endpoint call; keep the existing ≥70% messaging branch; show an error (not a fake score) if the AI service is down.
